## 三、甲机 (U1: AT89C52) 接线表

### 3.1 最小系统

| U1 引脚 | 连接 | 说明 |
|---------|------|------|
| **31 (EA/VPP)** | → VCC (+5V) | 使能内部程序存储器 |
| **40 (VCC)** | → VCC (+5V) | 电源 |
| **20 (GND)** | → GND | 地线（与乙机共地） |
| **18 (XTAL1)** | → 晶振一端 + 30pF→GND | 晶振电路 |
| **19 (XTAL2)** | → 晶振另一端 + 30pF→GND | 晶振电路 |
| **9 (RST)** | → 10KΩ→GND + 10μF→VCC | 复位电路（上电自动复位） |

**晶振电路接线**:  
- 放置 CRYSTAL，**双击**修改 Frequency 为 `11.0592MHz`
- XTAL1 → 晶振 Pin1 → CAP(30pF)→GND
- XTAL2 → 晶振 Pin2 → CAP(30pF)→GND

### 3.2 按键 K1、K2

| U1 引脚 | → | 元件 | 连接 |
|---------|---|------|------|
| **12 (P3.2/INT0)** | → | K1 一端 | K1 另一端 → GND |
| **13 (P3.3/INT1)** | → | K2 一端 | K2 另一端 → GND |

**注意**: 按键按下接 GND（低电平触发），可不加上拉电阻（51内部有上拉）。

### 3.3 串口连接

| U1 引脚 | → | 连接目标 |
|---------|---|----------|
| **10 (P3.0/RXD)** | ← | 乙机 U2 P3.1/TXD + VT2 TXD(经1KΩ) |
| **11 (P3.1/TXD)** | → | 乙机 U2 P3.0/RXD + VT1 RXD |

### 3.4 数码管

| U1 引脚 | → | 数码管引脚 | 电阻 |
|---------|---|-----------|:----:|
| **39 (P0.0)** | → 220Ω → | 7SEG Pin1 (a) | 可省略(仿真时) |
| **38 (P0.1)** | → 220Ω → | 7SEG Pin2 (b) | 同上 |
| **37 (P0.2)** | → 220Ω → | 7SEG Pin3 (c) |
| **36 (P0.3)** | → 220Ω → | 7SEG Pin4 (d) |
| **35 (P0.4)** | → 220Ω → | 7SEG Pin5 (e) |
| **34 (P0.5)** | → 220Ω → | 7SEG Pin6 (f) |
| **33 (P0.6)** | → 220Ω → | 7SEG Pin7 (g) |
| **32 (P0.7)** | → 220Ω → | 7SEG Pin8 (dp) |
| **1 (P1.0)** | → | 7SEG Pin9 (十位COM) | 共阳: 高电平亮 |
| **2 (P1.1)** | → | 7SEG Pin10 (个位COM) | 共阳: 高电平亮 |

> **关于 7SEG-MPX2-CA**:  
> - 共阳(CA) = Common Anode  
> - 位选高电平(1) = 该位点亮  
> - 段码低电平(0) = 该段点亮  
> - 段码表: `0xC0,0xF9,0xA4,0xB0,0x99,0x92,0x82,0xF8,0x80,0x90`

> **Proteus 引脚编号确认**: 放置元件后把鼠标悬停在每个引脚上，会显示引脚编号/名称。
> 7SEG-MPX2-CA 通常: Pin1=a, Pin2=b, Pin3=c, Pin4=d, Pin5=e, Pin6=f, Pin7=g, Pin8=dp, Pin9=COM1(十位), Pin10=COM2(个位)

## 四、乙机 (U2: AT89C52) 接线表

### 4.1 最小系统

同甲机：
| U2 引脚 | 连接 |
|---------|------|
| **31 (EA)** → VCC | **40 (VCC)** → VCC |
| **20 (GND)** → GND（与甲机共地） | |
| **18/19 (XTAL1/2)** → 11.0592MHz + 30pF×2 → GND | |
| **9 (RST)** → 10KΩ→GND + 10μF→VCC | |

### 4.2 按键 K3

| U2 引脚 | → | 元件 | 连接 |
|---------|---|------|------|
| **12 (P3.2/INT0)** | → | K3 一端 | K3 另一端 → GND |

### 4.3 LED

| U2 引脚 | → | 电阻 | → | LED | → |
|---------|---|:----:|---|-----|---|
| **27 (P2.6)** | → | 220Ω | → | LED1 阴极(短脚) | LED1 阳极(长脚)→VCC |
| **28 (P2.7)** | → | 220Ω | → | LED2 阴极(短脚) | LED2 阳极(长脚)→VCC |

> **LED 方向**: 低电平亮。阳极接 VCC，阴极经电阻接 MCU 引脚。

### 4.4 串口

| U2 引脚 | → | 连接目标 |
|---------|---|----------|
| **10 (P3.0/RXD)** | ← | 甲机 U1 P3.1/TXD |
| **11 (P3.1/TXD)** | → | 甲机 U1 P3.0/RXD + VT2 RXD |

## 五、VIRTUAL TERMINAL 接线

### VT1（监控甲→乙方向）

| VT1 引脚 | 连接 | 说明 |
|----------|------|------|
| **RXD** | → 甲机 U1 P3.1/TXD | 观察甲机发送的数据 |
| **TXD** | 悬空(NC) | 不发送 |

### VT2（监控乙→甲方向 + 发送指令）

| VT2 引脚 | 连接 | 说明 |
|----------|------|------|
| **RXD** | → 乙机 U2 P3.1/TXD | 观察乙机发送的数据 |
| **TXD** | → 通过1KΩ电阻 → 甲机 U1 P3.0/RXD | 可发送字符串指令给甲机 |

> **注意**: VT2 的 TXD 和乙机 TXD 都连到甲机 RXD（经 1KΩ 隔离），使用时要避免同时发送。

**VT 属性设置**: 双击 VT → 设置:
- **Baud Rate**: 9600
- **Data Bits**: 8
- **Parity**: NONE
- **Stop Bits**: 1
- **ASCII**: ✅

## 六、K1 操作说明

| 操作 | 方式 | 行为 | 对应实验 |
|:----:|:----:|------|:--------:|
| **短按 K1** | 按下<1s 松开 | 发送 `0x01` → 乙机 LED1亮，LED2灭 | 必做(1) |
| **长按 K1** | 按住≥1s | 发送 `"Hello 齐继浩 094424128\r\n"` | 选做(1) |

## 七、Proteus 操作步骤

### 7.1 新建项目
1. 打开 ISIS → **File** → **New Project**
2. 项目名: `串行通信` → 保存到 `Proteus仿真/` 目录
3. **Do not create PCB layout** → Next → **Finish**

### 7.2 放置元件
1. 左侧工具栏点击 **Component Mode (蓝色芯片图标)**
2. 点击 **P (Pick from libraries)**
3. 搜索并放置上表所有元件

### 7.3 连线
按上述接线表逐一连接（参考 `串行通信电路图.png`）

### 7.4 加载程序
1. **双击** 甲机 AT89C52 (U1)
2. **Program File**: 选择 `master_A.hex`（在 Keil 中编译生成）
3. **双击** 乙机 AT89C52 (U2)
4. **Program File**: 选择 `slave_B.hex`

### 7.5 运行
1. 点击左下角 **Play (▶)** 按钮开始仿真
2. 按下 K1/K2/K3 观察效果
3. 打开 VT1/VT2 观察串口数据

## 八、调试方法

### 查看串口数据
1. 仿真运行后，**双击** VIRTUAL TERMINAL 打开终端窗口
2. 甲机发送的数据会显示在 VT1 中
3. 乙机发送的数据会显示在 VT2 中

### 测试选做(2)
1. 在 VT2 终端窗口中**直接键盘输入**:
   - 输入 `11` → 乙机 LED1 亮
   - 输入 `10` → 乙机 LED1 灭
   - 输入 `21` → 乙机 LED2 亮
   - 输入 `20` → 乙机 LED2 灭
2. VT2 发送的字符会通过 TXD 经 1KΩ 电阻送到甲机 RXD
3. 甲机收到后原样转发给乙机（通过串口中断），乙机解析字符串指令

### 常见问题
| 问题 | 解决方法 |
|------|----------|
| 仿真不运行 | 检查晶振频率是否为11.0592MHz |
| 按键无反应 | 检查按键是否接 GND，是否用了外部中断引脚(P3.2/P3.3) |
| 数码管不亮或不正常 | 检查 P0 和 P1.0/P1.1 连接；检查共阳/共阴设置 |
| 串口通信无数据 | 检查 TX/RX 是否交叉连接；检查波特率设置 |
| LED 不亮 | 检查引脚是否正确；LED 方向（阳极→VCC，阴极→MCU） |

## 九、Keil 编译说明

1. 打开 Keil uVision 5
2. **Project** → **Open Project** → 选择 `master_A/master_A.uvproj`
3. **Project** → **Rebuild all target files**
4. `master_A.hex` 生成在 `master_A/` 目录下
5. 同样步骤编译 `slave_B/slave_B.uvproj`

> 如编译报错 "未找到 REG51.H"，检查 Keil 安装路径是否包含 C51 库文件。
